## Future Features & Roadmap
Here’s a prioritized list of potential improvements for **NukeCodeBridge**:
### 🔴 High Priority (Next Updates)
 * **Expanded File Support:** Support for .nk, .gizmo, .json, .yaml, and .md files.
 * **Enhanced UX:** Improved confirmation dialogs and clearer status bar feedback.
 * **Safety First:** Automatic backup creation (.bak) before overwriting existing scripts.
### 🟡 Medium Priority
 * **Studio Protection:** Read-only mode for "Official" repositories to prevent accidental edits to pipeline tools.
 * **Organization:** Script **categories and tagging** system with a dedicated filter UI.
 * **History:** A "Recent Files" list (last 10 opened) for quick access to frequent tasks.
 * **Auto-Save:** Optional background saving when changes are detected in the editor.
### 🟢 Nice-to-Have / Long Term
 * **Deep Search:** Search **inside** script content, not just the filenames.
 * **Versioning:** Simple version history with timestamps (e.g., tool_v001.py, tool_v002.py).
 * **Archive Tools:** Export/Import entire user repositories as .zip files for easy sharing or migration.
 * **Interactive Console:** A mini output console within the UI to view print() results and errors.
 * **Drag & Drop:** Drag any file directly into the editor to load it.
### 🚀 Advanced Pipeline Ideas
 * **Scene Support:** Previewing and loading nodes directly from .nk files.
 * **Collaboration:** A "Comments" section for shared scripts to allow peer feedback.
 * **Approval Workflow:** A "Request Review" status for scripts intended for studio-wide use.
 * **Execution Logging:** Track which scripts are used most frequently to identify candidates for permanent pipeline tools.
**Contributions welcome!** If you have ideas or want to help implement any of these features, feel free to open an issue or submit a pull request.
